## きっかけ

「MapLibreなら地図をグリグリ動かせるんじゃないか？」

それが最初の直感だった。MapLibre GL JSはWebGLベースのレンダリングエンジンで、大量のデータを乗せても60fpsでスムーズに動く。これで都道府県間の人口移動を可視化したら面白いBIになるんじゃないか、と思い立った。

参考にしたのは[このポスト](https://x.com/tsurezure_lab/status/2056801184737063362?s=20)。人口移動をフローとして地図に描くというアイデアに感動した。

---

## e-Statという宝の山

日本には[e-Stat](https://www.e-stat.go.jp/)という政府統計ポータルがある。住民基本台帳人口移動報告として、どの都道府県からどの都道府県へ何人移動したかが、**2020〜2023年分まるごとAPIで取れる**。しかも無料。

47都道府県 × 47都道府県の移動マトリクスが4年分。これがオープンデータとして公開されているのがすごい。

e-StatのAPIをMCPサーバーとして公開する方法については、[こちらの記事](https://note.com/clean_lynx1895/n/n007cdb3dac0c)を参考にした。

- 都道府県を選ぶと、その県との人口移動フローが矢印で表示される
- 青 = 転入超過（その県に人が来ている）、赤 = 転出超過（出ていっている）
- 矢印の太さ・透明度は転出入の差（ネット）に比例
- 矢印の上を粒子が流れるアニメーション
- 年度フィルター（2020〜2023 / 全期間）
- 「東京を除外する」チェックボックス（東京が支配的なので、他県の比較に使う）

## Claude Codeの/goalコマンドでワンショット生成

このアプリは**Claude Codeの `/goal` コマンドほぼ一発で生成**している。

`/goal` はゴールを自然言語で記述するとClaude Codeが自律的にタスクを分解・実行するコマンド。プロンプトに仕様を書いて投げたら：

1. e-Stat MCPサーバー経由でデータ取得
2. 47都道府県分の座標を国土地理院Geocoding APIで取得
3. MapLibre + Canvas 2Dオーバーレイの実装
4. 粒子アニメーションのロジック
5. UI（セレクター、年度ボタン、統計パネル、ランキング）

…まで自律実行して、単一HTMLファイルとして出力してきた。

自分がやったことはほぼ「仕様を書く」と「方針を変えるとき指示する」だけ。

## 技術スタック

| 技術 | 用途 |
|------|------|
| MapLibre GL JS v4 | WebGLによる高速地図レンダリング |
| 国土地理院タイル（blank） | 白地図ベースレイヤー |
| Canvas 2D API | フロー線・矢印・粒子のオーバーレイ描画 |
| Bezier曲線 | 都道府県間フロー線 |
| requestAnimationFrame | 60fps粒子アニメーション |
| e-Stat API | 住民基本台帳人口移動報告（2020〜2023） |
| FastMCP + Python | e-Stat APIをMCPサーバーとしてClaudeに公開 |
| Claude Sonnet 4.6 | コード生成 |

地図タイルはMapLibreのデフォルト（Maptiler等）ではなく**国土地理院の白地図**を使っている。余計な情報がなくフローの視認性が上がる。

## 感想

- **オープンデータがすごい**：47×47の移動マトリクスが4年分タダで取れるのは純粋に感動した
- **MapLibreがすごい**：WebGLなので大量フローを描いても落ちない、カスタムレイヤー自由自在
- **BIってすごい**：数字の表よりも地図で見た瞬間に「東京一極集中」が直感的に分かる
- **/goalがすごい**：仕様さえ書ければ実装はAIが走る。BI作成のハードルが劇的に下がった
